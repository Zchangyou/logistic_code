- **项目名称**：产业链供应链风险识别与智能处置系统

- **包管理**：pip + requirements.txt（或 poetry）

- **验证案例**：汽车发动机供应链（详见"开发思路.md 附录A"）

## 三、编码规范

### 3.1 命名规范
- **文件名**：小写下划线（`snake_case.py`）
- **类名**：大驼峰（`SupplyChainNetwork`）
- **函数/变量**：小写下划线（`calculate_betweenness`）
- **常量**：全大写下划线（`MAX_SUPPLY_TIER = 5`）

### 3.2 代码风格
- 遵循 PEP 8 规范
- 单个函数不超过 50 行，超过则拆分
- 类型注解：所有公开函数必须添加参数和返回值类型注解
- 文档字符串：所有公开类和函数使用 Google 风格 docstring

### 3.3 模块间接口规范
- 模块间通过明确定义的数据类（`dataclass` 或 `Pydantic BaseModel`）传递数据
- 网络模型输出标准化的 `NetworkData` 对象供风险模块使用
- 风险评估输出标准化的 `RiskReport` 对象供智能体模块使用
- 避免模块间直接访问内部实现，通过 `__init__.py` 导出公开接口

### 3.4 配置管理
- 所有可调参数（网络规模、风险阈值、模型超参数等）统一放在 `config/settings.yaml`
- 汽车发动机案例的专用参数放在 `config/case_auto_engine.yaml`
- 代码中不硬编码魔法数字，通过配置文件或常量引用

## 五、科研图表输出规范

### 5.1 图表格式要求
- **存储格式**：同时输出 PDF（矢量，用于论文/报告）和 PNG（300 dpi，用于 PPT 汇报）
- **输出路径**：`outputs/figures/阶段编号_图表名称.{pdf,png}`
- **字体**：中文使用 SimHei/Microsoft YaHei，英文使用 Times New Roman
- **配色方案**：统一使用项目配色，在 `src/visualization/style.py` 中集中定义

### 5.2 图表样式统一
- 所有图表通过 `style.py` 中的 `apply_research_style()` 函数统一设置
- 坐标轴标签、图例、标题均需中英文双语标注（中文在上，英文括号补充）
- 图表尺寸默认 `(10, 6)` 英寸，网络拓扑图 `(12, 10)` 英寸

## 六、Git 提交规范

```
<type>(<scope>): <subject>

feat(network): 实现参数化网络构造算法
fix(risk): 修复SIR模型传播概率计算错误
docs(agent): 补充RAG诊断流程说明
test(network): 添加拓扑分析单元测试
refactor(risk): 重构风险评分融合逻辑
```

- **type**：feat / fix / docs / test / refactor / chore
- **scope**：network / risk / agent / vis / sim / config

## 七、测试规范

- 每个模块核心逻辑需有单元测试
- 测试文件与源文件目录结构对应
- 运行方式：`pytest tests/ -v`
- 所有验证均基于汽车发动机案例的仿真数据执行

## 八、文档规范

- 每个模块的 `__init__.py` 包含模块功能简述
- 实验过程记录在 `notebooks/` 中，命名格式：`YYYY-MM-DD_实验描述.ipynb`
- 重要设计决策记录在 `docs/` 中
- 每阶段完成后在 `outputs/reports/` 中生成该阶段的结构化分析报告
