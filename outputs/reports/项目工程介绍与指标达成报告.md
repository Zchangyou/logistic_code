# 产业链供应链风险识别与智能处置系统

## 1. 项目工程概述

本项目面向科研场景，围绕汽车发动机供应链案例，构建了“网络建模 → 风险评估 → AI智能体处置 → 反馈更新”的闭环系统。  
工程按六阶段实现，核心模块如下：

- 模块2（`src/network/`）：供应链复杂网络建模、拓扑分析、脆弱性评估
- 模块3（`src/risk/`）：风险指标构建、SIR传播仿真、模糊+贝叶斯综合评估
- 模块4（`src/agent/`）：RAG瓶颈诊断、分级预警、策略推荐、多智能体协同处置

案例规模为 25 个节点、4 层级、38 条边，配套 6 期仿真数据和 4 类风险场景（芯片停供、稀土集中、区域中断、需求骤增）。

## 3. 设置指标达成情况（按《开发思路.md》对照）

### 3.1 功能指标

1. 功能(1)：多级网络建模 + 拓扑 + 脆弱性  
达成情况：**已实现**  
证据：阶段1/2报告完成 25 节点4层网络建模、拓扑指标、抗毁性与脆弱性分析，并输出对应图表 F1/F2。

2. 功能(2)：4类以上风险周期性识别  
达成情况：**已实现**  
证据：阶段3/4报告覆盖材料短缺、供应商集中、物流中断、需求波动 4 类风险；基于6期数据输出识别与评估结果。

3. 功能(3)：AI智能体自动生成防控策略  
达成情况：**已实现**  
证据：阶段5/6报告显示完成 RAG 诊断、预警、策略推荐与多智能体协同，形成 7 条综合处置动作并给出量化评分。

### 3.2 性能指标

1. 性能(1)：风险识别准确率 ≥ 90%  
达成情况：**已实现（超额）**  
结果：阶段4报告 `accuracy = 1.0`（100%），高于 90% 阈值。

2. 性能(2)：3 个智能体  
达成情况：**已实现**  
结果：阶段6报告 `agents_count = 3`（库存/物流/需求），并由协调智能体完成方案整合与冲突消解。

## 5. 复现实测说明（当前终端环境）

统一说明：本项目工程所有代码均在 Python 虚拟环境 `logistic` 中运行。

推荐启动方式（Windows PowerShell）：
`conda activate logistic`

本次在当前环境直接执行 `pytest -q` 时，出现两类环境问题：

- `ModuleNotFoundError: No module named 'src'`（测试运行路径/PYTHONPATH未配置）
- `ModuleNotFoundError: No module named 'pgmpy'`（依赖缺失）

这不影响仓库已有历史运行产出（`outputs/reports/` 与 `outputs/figures/`），但若需本机完整复现，应先补齐依赖并配置模块路径。
