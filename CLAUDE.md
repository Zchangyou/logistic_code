# 项目说明

## 项目概述

产业链供应链风险识别与智能处置系统。基于复杂网络建模、风险评估与 AI 智能体技术，实现供应链风险的识别、评估与自动化处置。本项目面向科研场景，所有功能需产出可用于论文和汇报的图表。

## 核心文件

- `研究内容和关键技术.md`：项目立项文档（只读参考，不可修改）
- `开发规范.md`：目录结构、编码规范、依赖库、图表输出规范
- `开发思路.md`：六阶段开发路线、验收标准、案例定义、仿真数据规格

## 三大模块

| 模块 | 目录 | 职责 |
|------|------|------|
| 模块2：网络建模 | `src/network/` | 供应链复杂网络构建、拓扑分析、脆弱性评估 |
| 模块3：风险评估 | `src/risk/` | 风险因素提取、SIR传播模拟、模糊+贝叶斯综合评估 |
| 模块4：AI智能体 | `src/agent/` | RAG瓶颈诊断、预警策略、多智能体协同处置 |

## 模块间数据流

```
NetworkData（网络模型输出）→ 风险模块输入
RiskReport（风险评估输出）→ 智能体模块输入
处置结果 → 网络模型更新（闭环）
```

## 验证案例：汽车发动机供应链

贯穿全部开发阶段的真实案例，详细定义见 `开发思路.md` 附录A：
- **规模**：25 个节点（12 原材料 + 9 零部件 + 3 集成商 + 1 总装厂）
- **仿真数据**：6 期周期性运营数据，预埋 8 个已知风险用于验证
- **风险场景**：4 个预设场景（芯片停供、稀土集中、华东区域中断、需求骤增）
- **案例数据路径**：`data/cases/auto_engine/`

## 技术选型

- Python 3.10+
- 网络：networkx
- 贝叶斯：pgmpy
- 模糊评价：scikit-fuzzy
- 科研绘图：matplotlib + seaborn
- LLM：通义千问（dashscope）
- RAG：langchain + faiss-cpu

## 图表输出要求

- 同时输出 PDF（矢量）和 PNG（300dpi），路径：`outputs/figures/`
- 中英文双语标注，统一样式由 `src/visualization/style.py` 管理
- 各阶段必需图表清单见 `开发规范.md` 第五节，共 20 张图表

## 开发约定

1. 开发前先阅读 `开发规范.md` 了解目录结构与编码规范
2. 每个阶段开发前先阅读 `开发思路.md` 对应阶段的内容
3. 模块间通过 dataclass / Pydantic 模型传递数据，不直接耦合内部实现
4. 所有可调参数放在 `config/settings.yaml`，不在代码中硬编码
5. 所有功能必须在汽车发动机案例上运行验证，不接受仅有代码无输出的阶段
6. 每阶段完成后需产出对应图表和结构化报告，存放在 `outputs/` 目录
7. 当前开发阶段：**阶段六**（多智能体协同与系统集成）
